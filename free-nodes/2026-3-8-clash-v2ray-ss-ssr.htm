在开始安装小猫咪Clash之前，我们需要做好以下准备工作：

第四章：核心配置指南

4.1 首次运行设置

初次启动小猫咪Clash时，界面可能会显得有些复杂。让我们一步步来：

  1. 语言设置：在"General"或"通用设置"中切换为中文界面（如果可用）
  2. 系统代理设置：决定是否让小猫咪Clash自动配置系统代理
  3. 开机启动选项：根据个人需求选择是否随系统启动

4.2 代理服务器配置详解

代理配置是小猫咪Clash的核心功能，这里我们详细讲解：

手动添加节点： 1. 进入"Proxies"或"代理"选项卡 2. 点击"Add"或"添加"按钮 3. 根据服务器提供的信息填写： - 名称：自定义便于识别的名称 - 类型：HTTP/HTTPS/SOCKS5等 - 服务器地址：IP或域名 - 端口：代理服务端口号 - 认证信息：用户名/密码（如有需要） 4. 高级选项可配置加密方式、协议参数等

订阅链接方式： 1. 在"Profiles"或"配置"页面 2. 点击"Download"或"下载"按钮 3. 粘贴订阅链接URL 4. 设置自动更新频率（建议24小时） 5. 保存后会自动获取节点列表

4.3 规则配置的艺术

小猫咪Clash的强大之处在于其灵活的规则系统：

基础规则设置： - DIRECT：直连，不经过代理 - REJECT：拒绝连接 - 代理名称：使用特定代理服务器

规则类型： 1. DOMAIN：基于完整域名匹配 2. DOMAIN-SUFFIX：域名后缀匹配 3. DOMAIN-KEYWORD：域名关键词匹配 4. IP-CIDR：IP段匹配 5. GEOIP：国家地区IP匹配

实用规则示例： - 国内网站直连规则：DOMAIN-SUFFIX,cn,DIRECT - 视频网站走特定代理：DOMAIN-KEYWORD,video,美国节点 - 屏蔽广告域名：DOMAIN-SUFFIX,ad.com,REJECT

FAQ

Sing-box 的 JSON 配置文件常见语法错误如何快速定位？
通过 JSON 校验工具或编辑器（例如 VSCode）打开配置文件，查看语法高亮与错误提示；也可运行 sing-box 并观察控制台错误信息，错误通常会指出第几行或哪个字段不合法以便修正。
如何优化 Clash 的 DNS 设置？
在 Clash 中使用自定义 DNS 可以加快域名解析速度。建议使用 doh（DNS over HTTPS）或 dot（DNS over TLS）配置方式，既能提升速度，也能保护隐私防止 DNS 泄露。
Hysteria2 的 QUIC 协议相比 TCP 有什么优势？
QUIC 协议基于 UDP，减少了握手延迟，支持多路复用和拥塞控制优化。Hysteria2 利用 QUIC 提供更低延迟、更高稳定性的网络连接，尤其适合高延迟或丢包环境。
如何用 systemd 管理 V2Ray/ Sing-box 服务？
创建 systemd 单元文件，指定 ExecStart 指令并设置 Restart=on-failure 与合适的 User/Group。启用服务并使用 `systemctl enable` 与 `systemctl start` 管理，便于开机自启与日志集中收集。
VLESS 的 XTLS 模式能提升传输效率吗？
XTLS 模式通过减少冗余加密和握手次数，降低延迟，提高吞吐量。在高并发和短连接场景下，效率明显优于普通 TLS，同时保持数据安全性和隐蔽性。
VMess 使用 AEAD 加密后连接性能如何？
AEAD 加密在保证数据安全的同时，性能开销较低，抗重放攻击能力强。高封锁环境下更稳定，同时减少流量特征泄露，适合长时间和高安全性需求的网络连接。
Matsuri 的延迟测试准确吗？
Matsuri 的延迟测试使用 ICMP 与 TCP 双重检测方式，可较准确反映节点实际响应时间。不过受网络波动影响，建议多次测试并结合速度测试结果综合判断。
V2Ray 的路由规则如何实现国内直连？
通过配置域名列表或 IP 段，将国内流量指向直连出口，而国外流量走代理节点。结合策略组和定期更新规则，可保持国内访问速度，同时优化海外访问体验。
VLESS + XTLS 模式在移动网络下表现如何？
